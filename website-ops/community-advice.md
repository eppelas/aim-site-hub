## 1. Claude Code + Obsidian как рабочая среда

- **Терминал с Claude Code прямо внутри Obsidian** через плагин Terminal — «Курсор для контента»: вся работа с текстами/ресёрчем без переключения окон. — Андрей Кирюшкин, 2025-12-15; Kirill Oleinichenko: «гениально, почему я сам не додумался». Позже Alex P нашёл более нативный плагин **obsidian-agent-client** (https://rait-09.github.io/obsidian-agent-client/) со скиллами и историей (2026-01-26); Николай Сенин советовал https://github.com/iansinnott/obsidian-claude-code-mcp (2025-12-25).
  *Для команды AIM: наш контент и так живёт в Obsidian-вольтах — дешёвый способ работать с ним агентно.*
- **CC (grep) vs Cursor (RAG) по базе заметок**: консенсус треда (2025-12-25, Кирюшкин / Яков Васильев / Сенин) — на размеченном вольте до ~5k заметок Claude Code с grep находит быстро и точно; на большой неразмеченной базе Cursor деградирует меньше. Разметка/naming — 80% успеха.
- **53 способа использовать Obsidian + Claude Code** — тред Steph Ango (CEO Obsidian): https://x.com/kepano/status/2007223691315499199, Николай Сенин обобщил в список (2026-01-03).
- **Бэкап вольта от «нейронка случайно затёрла»**: git в вольте + периодический дубль в S3 (Rainbow Keeper, Кирюшкин, 2025-12-17). Замечание: git не сохраняет ctime файлов при восстановлении.
- **Персональный «Obsidian-LMS» на Claude Code + Skills**: Сережа Рис за 5 вечеров и ~$150 токенов собрал систему «транскрипт лекции → атомы знаний → граф связей → генерация нового курса из атомов»; вся логика — в `.claude/rules/`, код только FastAPI+React (2026-01-01). Его вывод: «скиллы — новые программы, LLM — универсальный компьютер».

## 2. Скиллы, агенты, мультиагентность (Claude Code)

- **Наборы скиллов**: **superpowers** — «сильный буст продуктивности сессий» (Сережа Рис, Сенин, 2026-01-04); **OpenSpec** — «существенно снижает дрифт, крупные рефакторинги наконец доходят до конца» (Maxim j2h4u, 2026-01-04). Каталог SDD-инструментов: OpenSpec, spec-kit, BMAD-METHOD, nautex.ai (Maxim, 2025-11-13).
- **Скиллы вместо MCP**: скилл = пара строк frontmatter + progressive disclosure, не загрязняет контекст, в отличие от «жёсткого» MCP; в скилл можно завернуть скрипт-адаптер к БД/API и звать «дай сводку в X за Y» (Maxim + Сережа Рис, 2025-12-17). Пример с кодом: https://t.me/ris_ai/532. Оговорка Сенина: скиллы ненадёжны — модель может их проигнорировать, нужны детерминированные проверки.
- **Маркетплейс-репозиторий для скиллов**: свой репо со структурой `.claude-plugin/marketplace.json` (публичный + приватный) — подключаешь в Claude Code по ссылке на GitHub и получаешь авто-обновление и один консистентный набор скиллов/агентов на всех машинах. Пример: https://github.com/j2h4u/oh-my-claude-plugins (Maxim, 2026-01-25). Альтернатива — monorepo `docs/agent-skills`, из которого скиллы синкаются в конфиги всех агентов (CC/Gemini/Codex) — Ilya Gindin.
- **Дешёвые пары агентов в цикле**: cheap-code-writer + cheap-code-reviewer на Haiku, запускать максимум параллельно и крутить, пока ревьюер не даст добро (Maxim, 2026-01-22). Параллелизация задач: https://github.com/Yeachan-Heo/oh-my-claudecode (Rainbow Keeper).
- **Codex CLI + Claude Code в связке «один генерирует — другой верифицирует»**: Андрей Munтанион переписал проект ~200k строк с JS/TS на Go за 3 дня; также Conductor вместо консоли+VS Code, кастомные slash-команды (2025-12-24).
- **Sprint System: LMS за 3 дня** — Alex P (команда AIM): до 15 параллельных Claude-агентов, 41 спринт, 300+ задач, 64k строк кода; naming convention, компактизация контекста, передача задач между окнами. Презентация: https://lms-3-days-case.netlify.app/, видео: https://www.youtube.com/watch?v=mejkUQwcZLo (2026-01-29/31).
- **`/insights` в Claude Code** — анализирует, как ты его используешь, и советует, что вынести в CLAUDE.md для экономии токенов (Кирюшкин, 2026-02-05; подтвердили Rainbow Keeper, Кудряшов — результат проектно-специфичен).
- **Claude Code Router — сторонние модели в CC**: `ccr ui` → добавить модели → `ccr code`; под разные задачи разные модели (Сенин, 2025-11-15, инструкция в 5 шагов). Хит января: годовая подписка **GLM-4.7** (z.ai) — «ощущается как Sonnet 4.5», за 3 дня работы <10% пятичасового лимита, первая неродная модель, не рвущая агентный цикл CC (Сенин, Кирюшкин, 2026-01-02…21). На Cerebras GLM в 10 раз быстрее, но дороже и без кэша токенов.
- **RAG для Claude Code** (опенсорс-инструментарий): https://habr.com/ru/companies/redmadrobot/articles/982004/ (Maxim, 2026-01-03).
- **Мобильный клиент для Claude Code**: http://happy.engineering — open source, туннель к локальной машине из коробки (Maxim, 2025-11-15).
- **PAI/KAI Дэниела Мисслера** (автор fabric) — персональная ОС на Claude Code с авто-апдейтом и «дообучением» системы, есть в опенсорсе; видео: https://youtu.be/Le0DLrn7ta0 (Alex P, 2026-01-03; Кирюшкин: адаптирует с версии 1.0, самое полезное — детерминированные атомарные преобразования). Также воркфлоу Бориса Черни (автора CC): https://www.threads.com/@boris_cherny/post/DTBVlMIkpcm.
- **Обзор 30+ проектов автоматизации на Claude Code** (Кирюшкин, 2026-01-09): паттерн «life OS» = Claude Code + Obsidian + GitHub; нетехнические люди (юрист Anthropic, маркетологи) строят себе системы сами; шаблон Claudesidian.

## 3. Clawdbot / OpenClaw — Claude Code как Telegram-ассистент (тренд января 2026)

- **Суть**: open-source шлюз (https://docs.clawd.bot/, позже переименован в OpenClaw/MoltBot) — Claude Code живёт на VPS и общается с тобой в Telegram/WhatsApp/Slack; сам дописывает себе скиллы по ходу общения. Активно используют: Кирюшкин, Сенин, Alexey Ivanov (@ppprompt, итоги: https://t.me/ppprompt/2842), Ryspai, WhyNot, Alex P.
- Реальные кейсы: скилл расшифровки голосовых «создан с эллипсоида» одним сообщением (Сенин, 2026-01-19); подключение Garmin (body battery → прогноз продуктивности, готовая python-инструкция от Ryspai, 2026-01-26); ежедневные предложения бота «как меня улучшить» (Кирюшкин); Alex P подключил свои 37 скиллов из `.claude/` с умным меню по времени дня (2026-01-26).
- **Экономика**: API Opus — дорого (~15 центов за ответ). Решения: (1) Haiku — в ~60 раз дешевле; (2) подписка Claude Max через `claude setup-token` → `clawdbot models auth paste-token` (пошагово: Ryspai, 2026-01-26); (3) GLM-подписка как провайдер. От разлогина спасает heartbeat-скрипт по крону (Кирюшкин, 2026-01-28).
- **Синк Obsidian с ботом на VPS**: папка на VPS ↔ GitHub ↔ локальный вольт (Кирюшкин); либо Dropbox каждые 15 мин (Ivanov); либо syncthing.
- ⚠️ **Безопасность**: сотни развёрнутых инстансов торчат в интернет с дырами (пост Дениса, Shodan-поиск по порту 18789, 2026-01-27). Разворачивать только с авторизацией/за прокси.
- *Для команды AIM: прямой аналог наших ботов Сони/Суриката — стоит посмотреть на архитектуру скиллов и деплой.*

## 4. Контекст, базы знаний, память

- **Медальонная архитектура для личных данных** (bronze = сырьё: аудио/видео/pdf; silver = очищенное: транскрипты, парсинг, дедупликация; gold = готовые витрины под задачи) — подсмотрено у Databricks (Maxim, 2025-11-11). + YAML frontmatter как метаданные в md.
- **PDF → Markdown**: локальная **MinerU** (https://github.com/opendatalab/MinerU) — распознаёт формулы и таблицы, но слабовата на русском (Sergey Quantum); Chandra / LlamaParse (Maxim); хак — рендерить страницы в картинки и скармливать LLM как изображения (Vasilii Glebov, продовый опыт). Хранить исходники: «из них можно перегенерить всё, и качество генерации растёт каждый квартал».
- **Reposcan / ShotgunPro**: техническая карта проекта по C4 и доменам весом 10–20k токенов, подкладывается в каждый промпт — резко снижает галлюцинации агентов на большом коде (Alex Neighboor, Vanya Afanasov, 2025-11-14; shotgunpro.dev).
- **Vibecoding-starter и agents.md**: бойлерплейт для быстрого старта https://github.com/vasihc/vibecoding-starter; https://agents.md — единый формат инструкций для агентов (Alex P / Vasilii Glebov, лаба 2025-09-26).
- **Персональный супер-агент Севы Устинова** (Obsidian + Cursor, командный shared workspace): https://github.com/VsevolodUstinov/Personal-Super-Agent-Ru (шарили дважды, 2025-10-08).
- **Intention Management System Майка Яна (Manychat)**: OneSec (friction design вместо блокировок), voice → Obsidian через Mac mini-сервер, «alignment messages» — AI возвращает к намерениям после каждой заметки, Morning Brief + 5 сабагентов ночного процессинга (пересказ Alex P, 2025-12-04).
- **Самоулучшающиеся инструкции** (Rainbow Keeper, 2026-01-16): запрещено править результат руками — правишь инструкцию и перегенерируешь, затем просишь ИИ дописать в инструкцию «выученные уроки». Десятки циклов → инструкция собирает все грабли.
- **Knowlume** (Николай Сенин): веб-апп «отжима воды» — батч из 6 часов YouTube-видео → отфильтрованные фрагменты в базе знаний за 30–40 минут; рассказ: https://youtu.be/1SclrJKH1oQ (2025-10-31…12.2025).
- **Транскрипты YouTube массово**: бот @get_youtube_transcript_bot на youtube-transcript-api (Хабаров); скрипт на whisperx с конфигом плейлистов/каналов — «нейронка слушает конфы, пока я сплю», потом батчи в бесплатный web-Gemini (Sergey Quantum, 2025-10-24); бесплатный https://gemini-transcribe.fly.dev/ с таймкодами и диаризацией (Ilya Efimov).
- **Экспорт всех чатов ChatGPT** (Settings → Data export) как источник личного контекста для переноса в Claude Code (Ilya Gindin, 2026-01-29).
- **ActivityWatch** (https://activitywatch.net/) — бесплатный трекер времени по приложениям и даже по заметкам Obsidian; год статистики (Rainbow Keeper, 2025-12-30).
- **Локально vs VPS**: тренд — переносить агентов и данные на VPS (доступ с мобилки, бэкапы, stateless), синк кода через git, Obsidian — через родной E2E-шифрованный Sync/WebDAV; контраргумент — приватность личных дневников (тред 2026-01-25, Yury/Сенин/Alex S.).

## 8. Ресёрч и промптинг

- **parallel.ai** — deep research, который «упаковывает в таблицы под запрос, а не пишет диссертацию», хорошо по API; **exa.ai** — сильный семантический поиск (Яков Васильев, Артём Терёшин, 2025-12-24/25). Comet как основной браузер-агент.

Open source: github.com/trip2g/trip2g. Плюс «курилка агентов»: агенты ночью читают свой лог и лог соседа и адаптируют находки — скиллы не надо переносить руками.

- **Правила анализа интервью/качественных данных:** (1) не анализировать транскрипты пачкой — 50% ложных интерпретаций + гниющий контекст; (2) до анализа зафиксировать таксономию инсайтов и критерии подтверждаемости гипотез; (3) творческие доп-артефакты как стадия проверки («если видишь противоречие анализу — подсвети, не додумывай»).

## 7. Эвалы и бенчмарки

- **Свои бенчмарки вместо чуйки.** Минимальный старт: дать один живой инпут (например, транскрипт для разбиения по смысловым границам), прогнать 5 моделей через OpenRouter, сравнить → «настолько разные результаты, что чужие скиллы без бенчмарка теперь не беру». По итогам оставил gpt-5.4-mini на задачу. Отдельно: подробные MCP-инструкции делают дешёвые модели быстрее/пригоднее (статья trip2g.com/ru/thoughts/mcp_instructions_make_cheap_models_faster).
  Кто: Rainbow Keeper, 2026-07-07.

## 8. Безопасность и инфраструктура

- **Exec gate у Telegram-ботов-агентов.** Тест: отправить боту `запусти: python3 -c "print(...)"` — если выполнил, любой писавший боту может запускать код на сервере. Фиксы: allowlist ~25 безопасных паттернов, аналитические модели отрезаны от exec, опасные чат-команды отключены, фильтр prompt-injection, обёртка пересланных сообщений. Для Claude Code то же самое закрывают hooks + permission mode.
  Кто: Alexander Vasiliev, 2026-02-13 (markdown-гайд с copy-paste командами в чате).

- **Скилл хардening VPS.** После взлома VPS с агентами (DDoS с его машины, блокировка провайдером) собрал скилл, который анализирует и устанавливает протоколы безопасности сервера; «5000 попыток подбора пароля за день» — норма.
  Кто: Андрей Кирюшкин, 2026-03-13.

- **Проверка third-party скиллов:** Cisco skill-scanner (github.com/cisco-ai-defense/skill-scanner, работает в CI) — Maxim, 2026-02-16; security-скилл Амётова: каждый репо по 3 параметрам — закладки, дырявые зависимости, негативные отзывы на репо/автора (2026-06-15). Также: проверять зависимости после компрометации популярного плагина (reddit r/ClaudeAI, 2026-05-23).

- **Лимиты и мониторинг:** codexbar.app — аппка в статус-баре для лимитов Claude Code и Codex (Konstantin Popov, 2026-04-29); дни до сброса лимита в statusline (Ilya Golubev); VPS 48GB за ~$40/мес для агентов вместо апгрейда ноутбука — локально пара агентов, тяжёлое на сервере (Кирюшкин, 2026-07-04); шаринг подписок на команду через CLIProxyAPI (github.com/router-for-me/CLIProxyAPI) и производные (Maxim, 2026-06-10).
