Hard Rules

Эти правила нельзя восстанавливать из памяти чата или старых скриншотов.

common_content.mdreferences

Reference IDs

  • Reference-блоки заполняются только точными ID из свежего `common_content.md`.
  • Если сущности нет в экспортe, её надо создать в Studio и заново экспортировать файл.
  • Пустой reference-блок означает import behavior "all entities" и используется только осознанно.
product_codepayment

Payment Codes

  • `product_code` каждого тарифа приходит из каталога Дана / NocoDB.
  • Код не строится из `slug`, названия лаборатории или цены в описании.
  • Sanity хранит `productCode`; финальная EUR-цена приходит runtime-проверкой `/api/payment/*`.
slugdates

Lab Identity

  • Frontmatter `slug` задаёт URL и идентичность документа; имя файла не решает.
  • Даты пишутся строго `YYYY-MM-DD`.
  • FAQ остаётся плоским списком без категорий.

Creation Paths

Три поддержанных способа завести или обновить lab content.

Studiomanual

Manual Editing

  • Открыть AIM Studio.
  • Править `Лаборатория` или shared entities: speakers, cases, reviews, FAQ, quotes, philosophy, closing.
  • Publish запускает rebuild staging/new через Sanity webhook.
MD importagent

Markdown Upload

  • Скачать свежий `common_content.md` кнопкой `Экспорт общего контента`.
  • Заполнить `create-lab-from-md.md` по инструкции внутри шаблона.
  • В Studio использовать `Создать лабу из MD` или `Обновить из MD`.
gitimport workflow

Git Markdown Import

  • Положить файл в `content/labs/<slug>.md`.
  • Push в `main` запускает `.github/workflows/import-labs.yml`.
  • `scripts/import-lab.mjs` импортирует parsed lab в Sanity.
vibe codingpages

Content For Pages

  • Брать контент из Studio, Sanity export, `common_content.md` или tracked docs.
  • Не брать финальную истину из старого export-а, скриншота или пересказа чата.
  • Сохранять slugs, dates, reference IDs, product codes, CTA labels и payment links.

API And CLI

Подключение для локальной Studio, clean scratch Studio и repo import/export.

Commands

cd "Site Repo - ai-mindset-website" npm run dev # Astro, http://localhost:4331 cd studio && npm run dev # Sanity Studio, http://localhost:3343 npm create sanity@latest -- --project ms4mgrt9 --dataset production --template clean npm run export-lab <slug> npm run import-lab content/labs/<slug>.md

Environment

  • `SANITY_PROJECT_ID` for CLI import/export.
  • `SANITY_DATASET`, default `production`.
  • `SANITY_API_VERSION`, default `2026-03-29` in the repo script.
  • `SANITY_API_TOKEN` only for write/import contexts; never write temporary tokens into rules or tracked files.
  • The clean Sanity CLI bootstrap is scratch-only; do not run it inside the active repo unless replacing Studio is the task.
